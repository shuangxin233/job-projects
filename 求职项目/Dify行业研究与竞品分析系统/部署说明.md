# Dify行业研究与竞品分析系统 - 部署说明

## 项目定位

基于 Dify Workflow + FastAPI 的 Deep Research 类行业研究与竞品分析系统。后端支持本地知识库优先检索、证据登记、研究计划、信息缺口检查、报告质量评分、任务历史和 Markdown 导出。

## 跨设备运行结论

可以在其他设备运行。基础 FastAPI 后端可在未配置 Dify 的情况下生成本地草稿报告；完整工作流演示需要在 Dify 中导入 workflow，并配置 Dify API。

## 后端启动

```powershell
cd Dify行业研究与竞品分析系统\backend
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8010
```

可选 ChromaDB：

```powershell
pip install -r requirements-chroma.txt
```

可选 `.env`：

```text
DIFY_API_KEY=app-xxx
DIFY_WORKFLOW_URL=http://127.0.0.1/v1/workflows/run
DIFY_TIMEOUT_SECONDS=180
VECTOR_STORE_BACKEND=json
```

如需使用 ChromaDB：

```text
VECTOR_STORE_BACKEND=chroma
```

## 新设备演示流程

1. 启动 FastAPI 后端。
2. 调用 `/knowledge/documents` 或 `/knowledge/files` 添加本地资料。
3. 调用 `/knowledge/search` 验证 Top-K 检索。
4. 调用 `/research` 创建研究任务。
5. 通过 `/tasks` 查看历史任务。
6. 通过 `/reports/{task_id}.md` 导出 Markdown 报告。
7. 如配置 Dify，则先导入 `Deep Researcher On Dify .yml` 并按 `DIFY_WORKFLOW_UPGRADE_CHECKLIST.md` 检查节点。

## 注意事项

- 本目录不上传本地知识库向量数据，换设备后需要重新入库资料。
- Dify API 未配置时，系统会走本地 fallback draft，适合演示后端证据和质量控制链路。
- 完整项目效果依赖 Dify Workflow、模型服务和知识库资料质量。
