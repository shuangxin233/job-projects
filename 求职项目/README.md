# 求职项目

本仓库整理了 4 个面向求职展示的 AI / RAG 应用项目。每个项目都保留了可运行代码、依赖文件、核心说明和必要示例数据，适合在新设备上按说明安装依赖后复现主要流程。

## 项目列表

| 项目 | 类型 | 跨设备运行结论 |
| --- | --- | --- |
| AI私人厨师：家常菜推荐助手 | FastAPI + Next.js 家常菜推荐应用 | 可运行，Ollama 为可选增强 |
| 食谱知识库问答与推荐系统 | 本地 Markdown 知识库检索与推荐 | 可运行，仅依赖 Python |
| 企业知识库智能问答系统 | FastAPI + Streamlit + ChromaDB RAG 问答 | 可运行，需要配置大模型 API Key 并重新构建向量库 |
| Dify行业研究与竞品分析系统 | Dify 工作流 + FastAPI 研究报告生成 | 可运行，完整演示需要 Dify Workflow 和 API Key |

## 运行说明

每个项目目录中都提供了 `README.md` 或 `部署说明.md`。一般流程如下：

1. 克隆本仓库到新设备。
2. 进入对应项目子目录。
3. 按项目说明安装 Python 或 Node.js 依赖。
4. 如项目需要大模型、Dify 或第三方 API，复制 `.env.example` 并填写本地配置。
5. 启动 CLI、后端服务或前端页面，测试核心流程。

## 上传范围

仓库已排除以下不适合上传的本地文件：

- `.env`、`.env.local` 等真实密钥文件
- `node_modules`、`.next`、虚拟环境等构建或依赖缓存
- 本地聊天记录、临时文件和安装包
- 已生成的向量库持久化目录

企业知识库和行业研究类项目在新设备上运行时，需要重新上传资料或添加本地知识，系统会重新生成本地检索数据。
