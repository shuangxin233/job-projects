# 求职项目

本目录按简历项目名称整理了四个求职项目。每个子目录保留可运行源码、依赖文件、关键说明和必要示例数据，已排除本地虚拟环境、构建缓存、安装包、聊天历史、向量库持久化数据等不适合上传 GitHub 的文件。

## 项目列表

| 项目 | 是否可在其他设备运行 | 运行前置条件 |
| --- | --- | --- |
| AI私人厨师：家常菜推荐助手 | 可以 | Python、Node.js；Ollama 可选，不启动也有规则兜底 |
| 食谱知识库问答与推荐系统 | 可以 | Python；已保留运行所需 Markdown 菜谱语料 |
| 企业知识库智能问答系统 | 可以 | Python；需要按需配置 Groq 或 Gemini API Key，并重新上传文档生成本地向量库 |
| Dify行业研究与竞品分析系统 | 可以 | Python；完整 Dify 调用需要 Dify Workflow 和 API Key，未配置时后端可走本地草稿流程 |

## 运行说明

每个项目目录内都有 `部署说明.md`，优先按该文件启动。一般步骤是：

1. 克隆仓库到新设备。
2. 进入对应项目子目录。
3. 按 `部署说明.md` 安装 Python 或 Node 依赖。
4. 如项目需要大模型、Dify 或 API Key，复制 `.env.example` 并填写本地配置。
5. 启动后端、前端或 CLI，再按说明测试核心流程。

## 注意事项

- 本目录不包含真实密钥、`.env`、本地向量库、上传文件和聊天历史。
- 企业知识库类项目换设备后需要重新上传文档并构建向量库。
- Dify 项目如果要完整演示，需要先在 Dify 中导入 workflow，并配置 `DIFY_API_KEY` 与 `DIFY_WORKFLOW_URL`。
- 项目二为求职演示主线，不上传完整 all-in-rag 教程仓库，只保留 `job_project` 和运行所需菜谱 Markdown 语料。
