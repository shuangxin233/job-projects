# 企业知识库智能问答系统 - 部署说明

## 项目定位

面向企业文档的 RAG 智能问答系统，支持 PDF/TXT/Markdown/DOCX 上传、ChromaDB 向量检索、BM25/RRF 混合召回、引用来源返回和检索链路调试。

## 跨设备运行结论

可以在其他设备运行，但需要重新安装依赖并重新上传文档构建本地向量库。本目录不上传本机已生成的 Chroma 向量库和临时上传文件。

## 后端启动

```powershell
cd 企业知识库智能问答系统\server
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
uvicorn main:app --reload
```

`.env` 中按需配置：

```text
GROQ_API_KEY=your-groq-key
GOOGLE_API_KEY=your-google-key
```

## 前端启动

```powershell
cd 企业知识库智能问答系统\client
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

## 新设备演示流程

1. 启动后端和前端。
2. 在前端选择模型供应商。
3. 上传 PDF/TXT/Markdown/DOCX 文档。
4. 等待后端切片、embedding 和写入 ChromaDB。
5. 提问并查看 `answer` 和 `sources`。
6. 使用 Inspector 或 `/vector_store/search` 检查检索片段、来源和分数。

## 注意事项

- 如果不配置 API Key，模型调用会失败；但源码和接口结构仍可查看。
- 换设备后必须重新上传文档，因为本地向量库没有随 GitHub 上传。
- 不要提交 `.env`、本地向量库、临时上传文件和日志。
