# 食谱知识库问答与推荐系统

## 项目定位

这是一个面向求职展示的轻量级知识库问答与推荐项目。系统读取本地 Markdown 食谱语料，根据用户问题进行问题路由、关键词检索、证据召回和结果排序，最后输出带来源路径的 Top-K 食谱证据。

项目重点不是展示完整大仓库，而是把“知识库问答系统”的核心流程整理成一个可以在普通电脑上直接运行、也方便面试讲解的主线版本。

## 核心功能

- 读取 `data/C8/cook` 下的 Markdown 食谱语料。
- 自动解析食谱标题、类别和正文内容。
- 根据问题意图选择检索策略。
- 对标题、正文、类别进行打分排序。
- 返回 Top-K 相关食谱证据片段。
- 输出来源文件路径，方便追溯答案依据。
- 为后续接入 LLM、向量检索或 Graph RAG 保留扩展空间。

## 完整流程

```text
用户输入问题
-> 加载本地 Markdown 食谱知识库
-> 解析每篇食谱的标题、类别、正文
-> route_query() 判断问题类型
-> 根据问题类型选择检索策略
-> tokenize() 对问题和文档做关键词切分
-> score_doc() 计算标题、正文、类别和策略加分
-> 按分数排序并召回 Top-K 食谱
-> 输出证据编号、分数、来源路径和内容片段
```

## 检索策略

系统目前保留三类轻量策略：

| 策略 | 适用问题 | 处理方式 |
| --- | --- | --- |
| `hybrid-basic` | 普通菜名、食材、推荐类问题 | 主要根据标题和正文关键词匹配召回 |
| `hybrid-detail` | 做法、步骤、材料类问题 | 提高标题和正文内容的综合匹配权重 |
| `graph-lite-router` | 搭配、替代、适合、原因、对比类问题 | 使用更偏关系推理的规则信号进行扩展匹配 |

当前版本是轻量检索增强问答，不依赖大模型 API，也不强制依赖向量数据库。它可以作为后续升级到 Embedding、ChromaDB、Top-K 语义检索和 LLM 生成答案的基础版本。

## 运行方式

进入项目目录：

```powershell
cd 求职项目\食谱知识库问答与推荐系统
```

执行问答：

```powershell
python job_project\app.py --query "宫保鸡丁怎么做" --top-k 3
```

也可以询问搭配或推荐类问题：

```powershell
python job_project\app.py --query "鸡肉和蔬菜能搭配做什么" --top-k 5
```

输出内容包括：

- 使用的检索策略
- 策略选择原因
- Top-K 证据编号
- 食谱标题
- 分类信息
- 匹配分数
- 来源 Markdown 文件路径
- 证据预览片段

## 跨设备运行结论

可以部署到其他设备上运行。该项目不需要数据库服务、不需要大模型 API，也不需要前端服务；新设备只要安装 Python，并保留 `job_project` 和 `data/C8/cook` 目录，即可运行命令行检索。

## 面试讲解重点

- 这是一个本地知识库优先的问答系统主线。
- 用户问题不会直接丢给大模型，而是先经过问题路由和知识库检索。
- 系统先从本地食谱语料中找证据，再把证据作为可追溯上下文输出。
- `route_query()` 使用规则判断问题类型，不依赖 LLM。
- 当前检索以关键词匹配为主，属于轻量混合检索雏形。
- 如果继续升级，可以把 `score_doc()` 替换为向量相似度检索，接入 ChromaDB 或 Milvus。
- 如果接入 LLM，可以把 Top-K 证据作为上下文，让模型生成带引用编号的最终答案。

## 可扩展方向

- 增加 Embedding 向量化。
- 使用 ChromaDB、Qdrant 或 Milvus 保存向量。
- 使用语义相似度召回 Top-K 文档。
- 引入 BM25 + 向量检索的混合召回。
- 接入 LLM，根据证据生成自然语言答案。
- 增加 Web UI 或 FastAPI 服务。
- 为菜谱食材、口味、烹饪方式构建轻量知识图谱。
