# 简历和面试话术

## 简历项目名称

AI 私人厨师：基于 Next.js、FastAPI 和 Ollama 的家常菜推荐助手

## 简历项目描述

可以写成：

> 基于 Next.js + FastAPI 开发 AI 私人厨师应用，支持用户输入食材后生成家常菜推荐。后端通过 StreamingResponse 实现流式输出，优先调用本地 Ollama 模型生成结果，并提供规则菜谱兜底策略提升演示稳定性。项目支持会话历史保存、图片上传预签名接口和前后端分离调用。

## 简历技术点

- 使用 Next.js + React + TypeScript 实现聊天式交互页面。
- 使用 FastAPI 设计聊天接口、历史消息接口和 OSS 预签名接口。
- 使用 StreamingResponse 实现 AI 回复的流式返回。
- 接入 Ollama 本地大模型，支持通过环境变量切换模型。
- 使用本地 JSON 按 `thread_id` 保存会话历史。
- 设计模型不可用时的兜底菜谱逻辑，保证项目可演示。

## 面试 1 分钟介绍

> 这个项目是一个 AI 私人厨师应用，主要解决用户不知道现有食材能做什么菜的问题。前端是 Next.js 聊天页面，用户输入食材后调用 FastAPI 后端。后端会把用户消息和历史上下文组装成提示词，优先请求本地 Ollama 模型，并通过 StreamingResponse 把结果实时返回给前端。如果本地模型没有启动，系统会走内置菜谱规则兜底，保证项目可以稳定演示。这个项目让我实践了前后端联调、流式响应、本地大模型接入和会话状态管理。

## 面试问题与回答

### 为什么用流式响应？

普通接口要等 AI 全部生成完才能返回，用户等待时间比较长。流式响应可以让后端边生成边返回，前端边接收边显示，体验更接近真实 AI 产品。

### 前端怎么接收流式内容？

前端在 `lib/api.ts` 里使用 `response.body.getReader()` 读取后端响应流。每读到一段文本，就调用 `onChunk`，把这一段追加到当前 AI 消息气泡里。

### 后端怎么实现流式内容？

后端在 `chat.py` 里定义了 `generate()` 生成器。它会遍历 Ollama 返回的内容片段，然后 `yield` 给 FastAPI 的 `StreamingResponse`。

### Ollama 没启动时怎么办？

后端会捕获连接失败异常，调用 `fallback_recipe_answer()` 返回内置菜谱。这样项目在没有模型环境时也能演示基础链路。

### 为什么历史消息先用 JSON 保存？

这个项目定位是初学者求职项目，先用 JSON 可以降低环境复杂度，把重点放在 AI 调用、流式响应和前后端联调上。后续可以升级到 SQLite 或 PostgreSQL。

### 这个项目还能怎么优化？

可以从四个方向优化：第一，把 JSON 换成数据库；第二，增加用户登录和多会话列表；第三，接入图片识别模型；第四，增加菜谱收藏、购物清单和营养估算。
