# 学习手册：从零理解 AI 私人厨师项目

## 第 1 步：先理解项目解决什么问题

项目不是单纯聊天机器人，而是一个垂直场景应用：用户不知道冰箱里的食材能做什么，系统帮助用户生成菜谱建议。

你面试时要先讲业务价值：

- 用户输入食材，系统推荐菜。
- 推荐结果包含难度、时间、步骤、调料和安全提醒。
- 本地模型能保护隐私，适合家庭食材、饮食偏好这类个人信息。

## 第 2 步：理解整体流程

一次聊天的完整链路是：

```text
用户在前端输入食材
  -> 前端调用 POST /api/v1/chat/stream
  -> 后端保存用户消息
  -> 后端组装私厨提示词和历史消息
  -> 优先请求本地 Ollama
  -> 使用 StreamingResponse 一边生成一边返回
  -> 前端逐段追加到聊天气泡
  -> 后端保存 AI 回复
```

你要能说清楚三个关键词：

- `thread_id`：区分不同会话。
- `StreamingResponse`：让回答边生成边显示。
- `Ollama`：本地运行大模型，不依赖云端接口。

## 第 5 步：启动本地大模型

安装 Ollama 后执行：

```powershell

```

再测试：

```powershell

```

如果你想换模型，创建 `.env` 文件并写入：

```text
OLLAMA_MODEL=qwen2.5:3b
OLLAMA_BASE_URL=http://localhost:11434
```

## 第 6 步：理解后端代码

先看 `app/api/v1/chat.py`。

你要理解这几行逻辑：

- `store.add_message(...)`：保存用户消息。
- `build_ollama_messages(...)`：把系统提示词和历史消息组装成大模型能理解的格式。
- `stream_ollama_chat(...)`：调用 Ollama 的流式接口。
- `fallback_recipe_answer(...)`：Ollama 不可用时返回规则菜谱。
- `StreamingResponse(generate())`：把生成器里的内容逐段返回给前端。

再看 `app/services/recipe_engine.py`。

这里放了两类重要内容：

- `SYSTEM_PROMPT`：告诉大模型要扮演私人厨师，并规定输出结构。
- `INGREDIENT_RECIPES`：兜底菜谱库，保证无模型环境也能演示。

最后看 `app/services/chat_store.py`。

它负责把历史消息写入：

```text
app/data/chat_history.json
```

## 第 7 步：理解前端代码

先看 `私厨-前端源码/app/page.tsx`。

你要理解：

- `messages`：页面上显示的聊天消息。
- `processing`：控制发送按钮是否可用。
- `threadId`：当前会话 ID。
- `handleSend`：发送消息的主流程。
- `streamChat`：调用后端并读取流式响应。

再看 `私厨-前端源码/lib/api.ts`。

这里把后端接口封装成函数：

- `streamChat`
- `getChatHistory`
- `clearChatHistory`
- `uploadImageToOss`

## 第 8 步：面试时怎么讲技术亮点

建议讲 4 个亮点：

1. 流式响应：提升用户体验，不用等整段回答生成完。
2. 本地模型：通过 Ollama 接入本地 LLM，减少外部 API 依赖。
3. 兜底策略：模型不可用时仍能返回结果，演示更稳定。
4. 会话历史：用 `thread_id` 保存上下文，刷新后可以恢复历史消息。

## 第 9 步：你需要真正掌握的问题

面试官可能会问：

- 为什么要用流式响应？
- 前端如何逐段显示后端返回内容？
- 如果 Ollama 没启动怎么办？
- `thread_id` 是怎么起作用的？
- 为什么先用 JSON 存储，而不是一开始就上数据库？

你可以回答：

> 我先用 JSON 是为了降低初学阶段复杂度，把重点放在前后端联调、流式响应和 AI 调用链路上。后续如果做多用户或部署，就会替换成 SQLite 或 PostgreSQL。
